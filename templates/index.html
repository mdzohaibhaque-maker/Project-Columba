1. Evaluation Workflow

Determine code context first—this dictates how strict Code Execution rules should be.

Step 1: Determine Code Context (Crucial)

Before grading, classify the response into one of two contexts. This determines the strictness of the "Code Execution" rules.

Context A: Full Solution / Script

User Asked For: Task to be completed (e.g., "Write a script to scrape a website," "Create a snake game")

Standard: Code must be copy-paste executable.

  • All imports must be present
  • No undefined variables
  • No placeholder code that breaks
  • Script should run from start to finish
  • External resources (API keys, paths) may use placeholders
View Example 
import requests

def fetch(url: str) -> str:
    response = requests.get(url)
    response.raise_for_status()
    return response.text

if __name__ == "__main__":
    print(fetch("https://example.com"))

✔ Fully runnable • ✔ Contains imports • ✔ No missing definitions

Context B: Illustrative Snippet / Concept

User Asked For: Explanation or syntax example (e.g., "How does list.sort work?")

Standard: Code must be syntactically valid, but...

  • Imports may be omitted if they distract
  • Code does not have to be executable as standalone
  • Readability and conceptual accuracy matter more than completeness
View Example 
numbers = [3, 1, 2]
numbers.sort()
print(numbers)  # [1, 2, 3]

✔ Concept illustrated • ✔ No need for boilerplate • ✔ Syntactically valid

2. Dimension Rubrics & Edge Cases

Five independent evaluation dimensions with comprehensive specifications.

Key Point:

Each of the 5 dimensions below is graded independently. A response does NOT need to PASS all dimensions to receive credit—evaluate each separately based on its own criteria.

Dimension 1: Code Execution

Definition: The code must run without error in the expected context. Execution quality determines whether the code will execute as intended or fail due to syntax or missing components.

PASS Criteria

✔ Syntax Validity
Syntax must follow the rules of the language. Small format variations are fine, but missing punctuation or misalignment is not.

def add(a, b):
    return a + b

✔ Contextual Completeness (Full Solutions)
All imports, definitions, and dependencies must exist.

import json

def parse_json(text):
    return json.loads(text)
print(parse_json('{"x": 1}'))

✔ Runtime Stability
Code must not crash on typical input. Error handling not required unless part of instructions.

def safe_div(a, b):
    return None if b == 0 else a/b

✔ Environment Placeholder Safety
Placeholder values (e.g., API keys) must not cause runtime errors. Must be valid variable assignments.

API_KEY = "YOUR_API_KEY_HERE"
print("Using key:", API_KEY)
FAIL Criteria

✘ Syntax Error
Missing colons, mismatched parentheses, incorrect indentation.

def add(a, b)
    return a + b

✘ Hallucinated Libraries
Using imports or functions that don't exist in the language or ecosystem.

import superjson  # not a real library

✘ Fatal Omission (Full Solutions)
Essential imports or variables missing in a full script.

# Missing: import os
os.listdir(".")

✘ Broken Placeholders
Placeholder sections must not break execution. Leaving logic incomplete without safe guard is a failure.

def process_data(data):
    # TODO: implementation
process_data([1, 2, 3])  # Does nothing or errors

Execution Nuance Matrix

Scenario Verdict Reasoning
Illustrative snippet missing imports PASS Not required to be executable (Context B)
Missing imports in a full script FAIL User expects runnable code (Context A)
Pseudo-code requested N/A Not subject to execution rules
External API/DB placeholders PASS Allowed and expected
Plot-generating scripts PASS GUI output allowed if error-free
Dimension 2: Code Logic

Definition: Code should implement the correct algorithm and work as intended across normal and boundary cases.

PASS Criteria

✔ Correctness
Algorithm performs the correct task with accurate results.

def is_even(n):
    return n % 2 == 0

✔ Boundary Handling
Should handle empty lists, zero values, extremes when applicable.

def first(lst):
    return lst[0] if lst else None

✔ Safety
Should avoid unsafe operations such as division by zero where reasonable.

def div(a, b):
    return "error" if b == 0 else a/b

✔ Logical Flow
Clear and structured decision-making. Conditions properly ordered.

def grade(score):
    if score >= 90:
        return "A"
    elif score >= 80:
        return "B"
    return "C"
FAIL Criteria

✘ Incorrect Logic
Wrong algorithm, reversed conditions, flawed computation.

def is_even(n):
    return n % 2 == 1  # Wrong!

✘ Infinite Loops
Missing increments or broken loop conditions.

while i < 10:
    print(i)  # missing i += 1

✘ Silent Failures
Function appears to operate but does not update or return expected value.

def remove(lst, x):
    return lst  # doesn't actually remove!

Logic Nuance Matrix

Scenario Verdict Reasoning
Deprecated methods used PASS Code works logically despite old API
Off-by-one error FAIL Logical correctness is violated
Random/non-deterministic output PASS Random output not evaluated, only logic
Hardcoded dummy data PASS Allowed unless user provides real data
Dimension 3: Code Explanation

Definition: The explanation must accurately describe what the code does, why, and how.

PASS Criteria

✔ Alignment
Explanation should match the intent and function of the code exactly.

"This returns the square of a number"

def square(x):
    return x * x

✔ Prediction
If output is discussed, it must be correct.

"add(2, 3) returns 5"

def add(a, b):
    return a + b

✔ Context
Explanation matches the surrounding purpose.

"Reads file text"

with open("data.txt") as f:
    print(f.read())
FAIL Criteria

✘ Hallucination
Explanation claims functionality that does not exist.

"sends email" — actually does not

def notify():
    print("Email sent")

✘ Contradiction
Explanation and code disagree.

"returns a list" — but returns integer

def size():
    return 10

✘ Incorrect Output
Misleading output expectations.

"claims prints 1–5, but prints 0–4"

for i in range(5):
    print(i)  # 0, 1, 2, 3, 4

Explanation Nuance Matrix

Scenario Verdict Reasoning
Generic high-level explanations PASS "This sorts the list" is acceptable
Comment/text mismatch with code FAIL Comments must reflect actual behavior
Dimension 4: Code Style

Definition: Code should be readable, modern, clear, efficient, and maintainable.

PASS Criteria

✔ Naming
Variables & functions should be descriptive and easy to understand.

def calculate_total_price(items):
    pass

✔ Modern Conventions
Use PEP8 guidelines, f-strings, list comprehensions, meaningful spacing.

f"Hello {name}"  # f-strings
[x*x for x in range(10)]  # comprehensions

✔ Efficiency
Avoid unnecessary loops, avoid repeated operations.

[x*x for x in range(10)]  # Better than loop

✔ Formatting
Proper indentation, spacing, line breaks, type hints encouraged.

def add(a: int, b: int) -> int:
    return a + b
FAIL Criteria

✘ Obfuscation
Hard to interpret; unclear naming.

def f(a, b, c):
    return a*b+c

✘ Deprecated API Usage
Discouraged even if functional (e.g., pd.append in pandas 2.0+).

df = df.append(row)  # Deprecated!

✘ Needless Complexity
Overengineering simple logic.

def add(a, b):
    total = 0
    for _ in range(a): total += 1
    for _ in range(b): total += 1
    return total

Style Nuance Matrix

Scenario Verdict Reasoning
Code produces deprecation warnings FAIL Warnings indicate style/usage issues
Hardcoded constants in function FAIL Hides true logic or ignores input
Non-idiomatic but functional code PASS Allowed unless prompt requires idioms
Dimension 5: Coding Instruction Following

Definition: The code must follow the user's instructions exactly, respecting constraints, format, and intent.

PASS Criteria

✔ Follows Prompt Requirements
Required function names, signatures, and behavior must match.

"Write a function that adds one"

def add_one(x):
    return x + 1

✔ Constraint Respect
Must use or avoid specific structures (e.g., loops, recursion).

"Use a for loop"

for i in range(n):
    print(i)

✔ Formats Correctly
JSON strings, file output, or other formatting requirements must be honored.

import json
json.dumps({"env": "prod"})
FAIL Criteria

✘ Pivoting Topics
Answer must remain within the requested language or method.

Requested Python, provided JavaScript

✘ Violating Constraints
Using recursion when forbidden, using while instead of for, etc.

# "Don't use recursion" but code does:
def factorial(n):
    return 1 if n == 0 else n * factorial(n-1)

✘ Missing Key Components
Required calls or outputs missing.

def read(path):
    open(path)  # Never returns the content!

Intent Nuance Matrix

Scenario Verdict Reasoning
"Better but different" solution FAIL Must follow prompt exactly—even if "better"
Over-explaining code PASS Extra clarifying detail is allowed
Security refusal (harmful content) PASS Refusal is correct when appropriate

3. Common Failure Examples

Real-world scenarios demonstrating evaluation decisions.

Example 1: Missing Import (Full Solution)

Prompt: "Write a script to ping a server."

Response:

# Missing import os
def ping(host):
    response = os.system(f"ping {host}")
    if response == 0:
        print(f"{host} is up!")
    else:
        print(f"{host} is down or unreachable.")

def main():
    ping('127.0.0.1')

Analysis: This is Context A (full script), so imports are mandatory.

Scoring:

  • ❌ Execution: FAIL – Missing import os
  • ✅ Logic: PASS – Algorithm is correct
  • ✅ Style: PASS – Structure is reasonable
  • ⚠️ Explanation: N/A – No explanation provided
  • ❌ Intent: FAIL – Doesn't meet requirements (missing imports for full script)
Example 2: Broken Skeleton

Prompt: "Write a script to ping a server."

Response:

import os

def ping(host):
    # implementation goes here

def main():
    ping('127.0.0.1')

Analysis: Function body is empty, returns None. Calling it produces no output.

Scoring:

  • ❌ Execution: FAIL – Empty function creates logic error
  • ❌ Logic: FAIL – No implementation
  • ❌ Style: FAIL – Incomplete skeleton
  • ❌ Intent: FAIL – Doesn't solve the task
Example 3: Deprecated Method

Prompt: "How do you add a row to a pandas dataframe?"

Response:

import pandas as pd

def main():
    df = pd.DataFrame({'A': 2, 'B': 2})
    new_row = {'C': 5, 'F': 6}
    df = df.append([df, pd.DataFrame([new_row])], 
                   ignore_index=True)
    print(df)

Analysis: df.append() deprecated in pandas 1.4.0, removed in 2.0.0.

Scoring:

  • ✅ Execution: PASS (pandas < 2.0) / FAIL (pandas ≥ 2.0)
  • ✅ Logic: PASS – Row gets added correctly
  • ❌ Style: FAIL – Uses deprecated method
  • ⚠️ Intent: Depends on version requirement

Better approach: pd.concat([df, pd.DataFrame([new_row])])

Example 4: Constraint Violation

Prompt: "Find sum of a list WITHOUT using sum()."

Response:

def calculate_sum(numbers):
    """Find the total of all numbers."""
    return sum(numbers)  # Using built-in sum()

data = [1, 2, 3, 4, 5]
print(calculate_sum(data))  # Output: 15

Analysis: Directly violates explicit constraint "WITHOUT using sum()".

Scoring:

  • ✅ Execution: PASS – Code works
  • ✅ Logic: PASS – Produces correct answer
  • ✅ Style: PASS – Clean code
  • ❌ Intent: FAIL – Violates explicit constraint

Correct approach:

def calculate_sum(numbers):
    total = 0
    for num in numbers:
        total += num
    return total

5. Code Difficulty

Categorize coding prompts into difficulty levels based on required knowledge, complexity, and expected correctness rates.

Difficulty Level Definition Examples
Easy

A prompt is easy if:

  • • It requires basic knowledge of data structures of algorithms and limited domain knowledge or implementation context (architecture, libraries, pre-existing code)
  • • There is little ambiguity in the prompt (in case of underspecification, good default behaviors are easy to come up with or not important) and limited complexity of specifications (in terms of number of instructions)
  • • The solution is easy to explain and to debug, with a limited number of corner cases.
  • Expected correctness: Llama 3.2 3B should get about 30% of easy prompts completely correct.
  • • Implement the factorial function in Python.
  • • How can I rotate the x axis legend in matplotlib to write it vertically?
  • • In Bash, how do I list all text files in the current directory (excluding subdirectories) that have been modified in the last month?
Medium

A prompt is medium if:

  • • It requires mature knowledge of algorithms and data structures to get an optimal solution, or alternatively, knowledge of common libraries and concepts. May require additional code context.
  • • There is medium ambiguity in the prompt (e.g. needs to come up with reasonable ad-hoc data representation or class structure without explicit guidance). Multiple requirements should be satisfied or, in debugging problems, multiple bugs should be found.
  • • The problem involves corner cases that should be dealt with separately. Alternatively, the explanation of the solution requires some abstraction or decomposition of the problem into a few subproblems.
  • Expected correctness: Llama 3.1 70B should get about 30% of medium prompts completely correct.
  • • This function should return a list of lambda functions that compute successive powers of their input, but it doesn't work:
    def power_funcs(max_pow):
    return [lambda x:x**k for k in range(1, max_pow+1)]
    The function should be such that [h(2) for f in powers(3)] should give [2, 4, 8], but it currently gives [8,8,8]. What is happening here?
  • • Compute the longest increasing subsequence in a list. Implement the solution in C++.
  • • How can I use the Instagram Python API to create a post automatically?
Hard

A prompt is hard if:

  • • It requires expert domain knowledge or information on the specific application or deployment scenario, including substantial specific API/code context,
  • • Finding good solutions involves non-trivial design decisions regarding data structures, algorithms or code architecture/design patterns. Finding a solution could also require solving several non-trivial subproblems or, in the debugging case, finding non-trivial bugs.
  • • The problem may involve tricky corner cases and explaining the solution to a non-expert requires adding context.
  • Expected correctness: Llama 3.1 405B should get about 10% of hard prompts completely correct.
  • • What would be AdagradW, the equivalent of AdamW for Adagrad? Implement AdagradW in Python.
  • • Make a pong game in Rust.
  • • Implement a REST API that does account management for my website. The necessary functionality includes creating a user, fetching user information (username, full name, email address), updating a user's full name or email address, updating a user's password, and deleting a user. The functionality to fetch, update, or delete should be protected by an authentication scheme that verifies the user's password. Store all information in a sqlite table.
